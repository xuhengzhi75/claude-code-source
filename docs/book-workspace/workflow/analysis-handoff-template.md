# 源码分析交接卡模板

本模板用于“源码分析侧 -> 写作侧”的标准交接。目标是减少重复作业，让写作侧拿到材料后可以直接落正文。

## 模板

```md
# [章节名] 源码分析交接卡

## 1) 一句话结论
- 结论：<本章最核心判断>
- 状态：verified | inference

## 2) 本章最该讲的点（3~5条）
1. <机制或主流程>
2. <关键边界条件>
3. <失败/恢复/治理逻辑>
4. <设计取舍或约束>
5. <可选新增信息>

## 3) 不要重复讲的点
- 已在前章充分解释：<主题A>
- 本章只引用不重讲：<主题B>
- 容易同义重复的判断：<判断C>

## 4) 源码锚点
- 锚点：`<相对路径>`::<函数/类/分支/对象名>
- 状态：verified | inference

## 5) verified / inference 写法规范
- verified：源码显示…… / 在 `...` 的 `...` 分支中可见……
- inference：基于当前代码结构，推测/更像是……
- 禁止：把 inference 写成确定事实

## 6) 章节边界
- 本章覆盖：<范围>
- 本章不覆盖：<范围>
- 与相邻章节分界句：<1句>

## 7) 长度与信息密度要求
- 目标长度：<建议区间>
- 每段至少提供一个新信息单元
- 不用“本章将……”式空铺垫
- 不靠同义重复撑长度
- 若删 20%~30% 不影响理解，则视为不达标

## 8) 写作落地提示
- 推荐顺序：结论 -> 机制 -> 边界 -> 证据 -> 小结
- 读者定位：弱技术读者可读，但事实不降级
```

## 协作规则摘要
- 分析侧优先交付：一句话结论 + 关键锚点 + 不要重复讲的点
- 写作侧重点是表达与结构收束，不重复深挖同一批源码
- 每完成一点就立即 push，开始前先 pull
