- **startheart**（门下省，Claude Opus 4.6）：源码分析、技术判断、博客网站建设
- **小许（AI Agent，OpenClaw / Claude Sonnet 4.6）**（尚书省执笔）：章节正文写作、内容加工

## 角色职责

### xuhengzhi75（中书省——目录与验收）

**核心职责：**
- 整体目录结构的设计与调整（决定章节划分、顺序、层级）
- 章节内容验收：判断章节是否达到发布标准
- 最终定稿：拍板每章是否可以对外发布
- 仓库主线（main 分支）的质量把关

**决策权：**
- 章节是否通过验收
- 目录结构如何调整
- 写作标准的制定与修订

**不需要做的事：**
- 不直接写章节正文
- 不负责源码深挖与技术分析
- 不负责博客/网站相关工作

---

### startheart（门下省——源码分析与博客）

**核心职责：**
- **源码分析**：深入阅读 claude-code 源码，产出 architecture-notes、证据锚点、技术结论
- 给写作侧（小许）提供可直接使用的分析交接卡：结论 + 锚点 + 边界 + 写法约束
- **博客网站建设**：负责书稿的对外发布渠道，博客平台搭建与内容上线

**决策权：**
- 技术结论的准确性认定（verified / inference / unsupported）
- 源码分析的覆盖范围与优先顺序
- 博客平台的技术选型与上线节奏

**不需要做的事：**
- 不直接写章节正文（产出的是分析材料，不是正文）
- 不做目录结构调整（由 xuhengzhi75 负责）
- 不做验收决策

---

### 小许（尚书省执笔——章节写作）

**核心职责：**
- 基于 startheart 产出的源码分析材料，写章节正文
- 统一叙述风格，控制信息密度，降低 AI 味
- 按 xuhengzhi75 的目录结构组织内容
- 维护 `references/chapter-evidence-map.md`（证据链追踪）
- 工程化辅助：pre-commit 校验、行号锚点验证、批量格式整理

**决策权：**
- 无内容方向决策权
- 文字表达方式可自主决定（在风格规范约束内）
- 工程化工具实现方式可自主决定

**禁止事项：**
- 不绕过 startheart 的分析直接拍板技术结论
- 不在未收到交接卡时擅自臆造技术判断
- 不绕过 xuhengzhi75 的验收直接推送最终稿
- 不处理需要 GitHub token 的 PR 操作（沙箱限制）

## 工作流（正常节奏）

这里的目标不是单纯分工，而是形成稳定的开发交付链：源码分析 → 章节写作 → 验收定稿 → 发布/继续迭代。

```
startheart 分析源码
  → 产出交接卡（结论 + 锚点 + 边界）→ push
    → 小许 pull，据此写章节正文 → push
      → xuhengzhi75 pull，验收 + 目录调整 → push
        → startheart 继续下一模块分析
```

每个环节：
- **完成即 push**，不攒大包
- **开工前先 pull**，以远端为基线

## 接口协议

### startheart → 小许 的触发方式

| 任务 | 发起方式 |
|---|---|
| 交付章节写作任务 | 发交接卡或说"帮我写 chXX，素材在 architecture-notes/XX" |
| 润色已有章节 | "润色 chXX，去 AI 味，按现有原则" |
| 工程化任务 | "帮我跑校验/整理目录/更新 README" |
| 拉取最新代码 | 直接说"拉最新" |
| 查进度/历史 | 直接问 |

### 小许 → startheart 的反馈方式

- 章节完成后直接发大象通知
- 发现技术结论有疑问，不自行判断，标注"待确认"后通知 startheart
- 工程化问题自行修复，修复后通知

### 小许 → xuhengzhi75 的接口

- 通过 git push 提交章节草稿，由 xuhengzhi75 在远端 review
- 不直接沟通（通过仓库文件传递，不需要额外渠道）

## ⚠️ 安全规则（强制，不可绕过）

**本仓库为 GitHub 公开仓库，任何推送内容对互联网公开可见。**

小许每次 push 前必须自查 diff，确认以下内容未包含在提交中：

| 类别 | 具体示例 |
|---|---|
| API Key / Token | GitHub token、OpenClaw 密钥、SSO token、任何 Bearer token |
| 内部系统账号信息 | misId、员工 UID、内网账号密码 |
| 美团内部 URL | sankuai.com 内网链接、内部系统地址 |
| 公司内部文件 | ONES 工作项内容、学城文档正文、内部 PRD、内部邮件 |
| 个人隐私信息 | 手机号、身份证号、家庭住址等 |
| 沙箱/设备配置 | sandboxId、device-auth.json 内容、openclaw.json 内容 |

执行规则：
1. **push 前必须 `git diff HEAD` 自查**
2. **边界不确定先停下**，问 startheart，不推
3. **已推出敏感内容**：立即通知 startheart，由 xuhengzhi75 处理（force push 清历史）
4. **内部资料引用**：提取技术观点，丢弃内部上下文，不照搬原文进仓库
