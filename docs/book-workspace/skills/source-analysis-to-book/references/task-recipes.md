# Task Recipes

## Recipe 1: First contact with a new repository
1. Find entrypoints and top-level execution flow.
2. Locate state carriers, recovery logic, and tool boundaries.
3. Produce 3-5 architecture notes before drafting chapters.
4. Build a chapter evidence map before long-form writing.
5. Create status and handoff artifacts before ending the session.

## Recipe 2: Resume after interruption
1. Read `../../planning/project-status.md`.
2. Read `../../status/status-index.md` and the most recent `status-*.md`.
3. Read `../../references/chapter-evidence-map.md`.
4. Continue the highest-priority unfinished writing or analysis item.

## Recipe 3: Turn evidence into chapter structure
1. Start from a non-obvious chapter question.
2. Pull verified facts from evidence notes.
3. Separate mechanism, constraint, and effect.
4. Add edge cases and one non-obvious conclusion.
5. Check that the draft did not degrade into a module catalog.
