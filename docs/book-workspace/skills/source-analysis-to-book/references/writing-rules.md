# Writing Rules Reference

## Prefer
- control-flow over folder listing
- non-obvious questions over topic buckets
- edge cases over happy-path summaries
- mechanism + constraint + effect over generic praise
- source-backed claims over freehand conclusions

## Avoid
- module catalog writing
- repeated high-level AI clichés
- mixing verified facts with inference
- writing chapters before building an evidence base
- losing work continuity because status docs were skipped
