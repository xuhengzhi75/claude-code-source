# Workflow Reference

## Recommended Sequence
1. Scan repository entrypoints and primary execution paths.
2. Identify state carriers, recovery logic, and non-removable components.
3. Build evidence notes with verified facts and inference separated.
4. Convert findings into chapter questions or analysis sections.
5. Maintain status, handoff, and coverage artifacts for continuation.

## Required Output Properties
- Not a directory walkthrough
- Has explicit system tension or design tradeoff
- Marks evidence vs inference
- Preserves continuation state for later writing rounds
