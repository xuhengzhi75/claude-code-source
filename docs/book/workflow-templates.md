# Workflow Templates v0

## 1. 源码拆解到章节流程
1. 选定一个系统主题（如入口、工具、任务、恢复）
2. 先读相关源码文件和已有 architecture notes
3. 抽取 5-10 个可核对结论
4. 建立章节结论 ↔ 源码锚点映射
5. 改写为弱技术读者可读章节
6. 做一次 evidence review
7. 再做一次风格收敛

## 2. 新章节生产流程
1. 明确章节回答的问题
2. 列出 3-5 个核心结论
3. 为每个结论补源码锚点
4. 先写最小可读版
5. 补“为什么这对读者重要”
6. 做去重和术语统一
7. 补本章小结

## 3. 夜间小步快跑流程
1. 先确认本轮只推进一个小主题
2. 补一条可核对信号或一段正文
3. 更新 status 文档记录本轮动作
4. 检查 git 状态
5. 提交最小增量
6. 下一轮继续

## 4. 章节验收流程
1. 读者是否能在不懂源码的前提下读懂
2. 关键判断是否都能回到源码结构
3. 是否把 inference 当成 verified 写了
4. 是否存在重复、空话、模板腔
5. 是否给出了本章真正有用的结论
