# Writing Guide（统一入口）

更新时间：2026-04-01

本文件是书稿“写作规范层”的统一入口。目标是把分散在 `requirements.md`、`writing-principles.md`、`WORKING-METHOD.md` 的规则收敛成一套可执行、可验收、可协作的最小工作协议。

---

## 0. 这本书要写成什么样

一句话目标：**让非技术读者看懂 Claude Code 的系统级能力，并能迁移到自己的业务 Agent。**

核心约束：
- 读者导向：默认读者“可复制命令，但不深懂代码”。
- 机制导向：解释“为什么这样设计，不这样会怎样”。
- 证据导向：关键结论要能被源码复核。
- 简洁导向：高信息密度，禁止同义反复凑篇幅。

参考定位详见：`docs/book/requirements.md`

---

## 1. 五条写作红线（必须遵守）

### 1) 信息密度优先
- 每段至少提供一个新信息单元（机制、边界、证据、反例、取舍之一）。
- 能删 20%~30% 且不影响理解，视为不合格。
- 少过渡废句，少“重要的是…/下面来看…/综上所述…”。

### 2) `verified` / `inference` 严格分离
- **verified**：有源码锚点（文件 + 函数/分支/对象，关键点尽量带行号）。
- **inference**：基于证据的推断，必须显式标注，措辞保守。
- 不允许把推断写成事实。

### 3) 章节边界明确
每章开工前先写清三件事：
- 本章第一性问题（要解决的根本矛盾）
- 本章覆盖范围（讲到哪）
- 本章明确不讲（避免与相邻章节重叠）

### 4) 避免重复
- 不重复讲已达标章节的结论；只补“新增机制/新增证据/新增边界”。
- 不做目录树式模块罗列；只保留能支撑主张的最小结构信息。

### 5) 读者导向
- 用读者可理解的语言解释机制，但不牺牲正确性。
- 不靠夸张、比喻、情绪化措辞提高可读性。
- 始终区分“模型能力”与“系统能力”。

---

## 2. 标准章节骨架（推荐模板）

1. **核心主张（一句话，可被反驳）**
2. **第一性问题（为什么需要这个机制）**
3. **机制展开（关键决策 + 为什么不是替代方案）**
4. **证据锚点（verified）**
5. **边界与技术债（何时失效/何时退化）**
6. **可迁移原则（适用条件 + 反例）**
7. **本章不覆盖项（章节边界）**

> 骨架目的：保证“观点—证据—边界”闭环，而非平铺信息。

---

## 3. 章节验收清单（发布前自检）

### A. 基础通过线
- [ ] 有且仅有一个清晰核心主张（可被反驳）
- [ ] 关键结论具备源码锚点；无法验证处已标注 inference
- [ ] 解释了“为什么这样设计，不这样会怎样”
- [ ] 至少 1 个非显然发现
- [ ] 删除冗余后信息量未被掏空

### B. 质量增强线
- [ ] 说明了至少一个替代方案及未采纳原因
- [ ] 写明关键假设与适用条件（避免过度泛化）
- [ ] 指出边界条件或技术债
- [ ] 读者读完后需记住的核心概念 ≤ 3

详细高阶标准详见：`docs/book/writing-principles.md`

## 4. 协作与交接协议（分析侧 ↔ 写作侧）

分析侧交付最少包含：
1. 一句话结论
2. 本章最该讲的 3~5 点
3. 明确不要重复讲的点
4. 关键锚点（文件 + 函数/分支/对象）
5. 仅能写为 inference 的判断
6. 章节边界（讲到哪/不讲哪）

协作节奏：
- 开工前先 pull
- 完成一小段可验收成果就 push
- 轮动推进，不攒大包

详见：`docs/book/WORKING-METHOD.md`

---

## 5. 三个常见失败模式（写作时优先规避）

1. **结构转述型失败**：只说“有什么模块”，不解释“为何存在”。
2. **证据断裂型失败**：有观点无锚点，或有锚点无机制解释。
3. **边界失焦型失败**：章节之间互相抢内容，导致重复与信息稀释。

## 6. 文档关系图（如何使用）

- `writing-guide.md`（本文件）：统一入口、最小可执行规则。
- `requirements.md`：读者定位、全书目标、写作风格与密度底线。
- `writing-principles.md`：完整方法论与高强度验收条款。
- `WORKING-METHOD.md`：源码阅读顺序、协作流程、交接格式。

建议使用方式：
- 日常写作：优先遵循本文件 + `WORKING-METHOD.md`
- 章节终审：再对照 `writing-principles.md` 做强化验收

## 7. 一页执行版（最小记忆集）

只记 5 句话即可开始写：
1. 先写可反驳主张，再写内容。
2. 关键结论必须可复核；推断必须标注 inference。
3. 每段都要有新增信息，禁止重复改写同一判断。
4. 每章先划清边界，避免跨章抢内容。
5. 永远为目标读者写：易懂，但不失真。
